# 最小可落地设计

## 结论

第一版采用：

- 本地 Codex
- 一个独立 skill
- 明确的输入目录和输出目录

不采用：

- 接入 OpenClaw
- 改造主项目 `src/module/agent`
- 重做新的 agent runtime

## 目标

把“RSS 新闻 -> 机会分析 -> 结构化输出”跑通。

## 最小流程

1. 外部流程先把 RSS 或新闻摘要整理为文本或 Markdown 文件
2. Codex 读取 `inputs/` 下的内容
3. Codex 按 skill 规则提取：
   - 主题
   - 关键信号
   - 潜在机会
   - 风险
   - 下一步行动建议
4. 输出到 `outputs/`

## 输入约定

建议每次分析一个文件，例如：

- `inputs/2026-03-23-news.md`

文件内容建议包含：

- 标题
- 来源
- 时间
- 摘要
- 原文链接

## 输出约定

建议输出一个 Markdown 文件，例如：

- `outputs/2026-03-23-opportunities.md`

输出结构建议：

1. 今日主题概览
2. 单条新闻机会分析
3. 机会优先级排序
4. 风险与不确定性
5. 可执行行动清单

## 为什么这是最小方案

- 不碰主工程已有代码
- 不需要先解决主工程 Agent 层的历史问题
- 可以先验证“skill 是否真能稳定产出有价值结果”
- 失败成本低，成功后再考虑集成

## 后续升级路径

如果这一版有效，再考虑：

1. 增加输入标准化脚本
2. 增加结构化 JSON 输出
3. 增加自动批处理入口
4. 最后再评估是否并入主项目
